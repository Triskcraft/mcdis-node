Configuracion

El archivo default es config.json. La propiedad processes es un objeto donde cada clave es el nombre del proceso. Si un proceso no define cwd, McDis usa ese nombre como carpeta. 

{
  "$schema": "https://triskcraft.github.io/mcdis-node/config.schema.json",
  "discord": {
    "tokenEnv": "DISCORD_TOKEN",
    "panelChannelId": "123456789012345678",
    "prefix": "!!"
  },
  "mods": {
    "enabled": true,
    "directory": ".mdmods"
  },
  "processes": {
    "smp": {
      "type": "server",
      "startCommand": "java -Xms1G -Xmx1G -jar paper.jar nogui",
      "stopCommand": "stop",
      "plugins": {
        "enabled": true,
        "directory": ".mdplugins"
      }
    }
  }
}

Plugins Por Proceso

Los plugins viven en <cwd>/.mdplugins. Pueden ser archivos .js, .mjs, .cjs, .ts, o directorios con package.json y main. 

import { Plugin, type PluginContext } from 'mc-dis/plugin';

class AutoSavePlugin extends Plugin {
  name = 'auto-save';

  async onConsoleLine(line: string) {
    if (line.includes('Done')) {
      this.context.execute('save-on');
    }
  }
}

export async function load(context: PluginContext) {
  return new AutoSavePlugin(context);
}

Mods Globales

Los mods viven en .mdmods. Pueden acceder a McDisApp, escuchar eventos globales, registrar helpers e inyectar plugins en procesos. 

import { Mod, type ModContext } from 'mc-dis/mod';

class BroadcastMod extends Mod {
  async onLoad() {
    const unregister = this.app.registerHelper('broadcast', {
      broadcast: (message: string) => {
        for (const process of this.app.getProcesses()) {
          process.execute(`say ${message}`);
        }
      }
    });

    this.addCleanup(unregister);
  }
}

export async function load(context: ModContext) {
  return new BroadcastMod(context);
}

APIs Principales

  • McDisApp: procesos, mods, helpers, Discord y eventos globales.
  • MinecraftProcess: proceso local, stdin, lifecycle y eventos.
  • Plugin: extension por proceso.
  • Mod: extension global.

Comandos

  • !!start smp
  • !!stop smp
  • !!restart smp
  • !!plugins-reload smp
  • !!mods-reload

Eventos

MinecraftProcess y McDisApp son EventEmitter. Todo evento de un proceso se reemite en la app.

Proceso

process:started, process:stopped, console:line, discord:command, plugin:loaded.

Global

app:ready, app:shutdown, mod:loaded, helper:registered, process:event.
